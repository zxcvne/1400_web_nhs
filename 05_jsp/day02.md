1. include 디렉티브 태그
    <%@ include file="파일경로" %>

    현재의 jsp의 include가 있는 영역에 외부 파일의 내용을 포함
    가장 많이 사용하는 것은 header, footer => 페이지마다 동일
    유지보수+편의성을 위해 외부 파일로 만든 후 include
    => 모듈화

    * 모듈화는 1회성이 아닌 재사용 목적으로 구성하는 것

2. 액션 태그(Action Tag)
    서버 또는 클라이언트에게 어떤 행동을 하도록 명령
    다른 페이지의 실행 결과를 현재 페이지에 포함
    자바 빈즈 등의 다양한 기능 제공
    액션 태그의 형식은 XML 형식이며 <jsp: ________ />

    1) forward : <jsp:forward />
        다른 페이지로 이동
    2) include : <jsp:include />
        외부 페이지의 내용 포함
    3) param : <jsp:param />
        현재 페이지에서 다른 페이지에 정보 전달
    4) useBean : <jsp:useBean />
        Java 파일에 있는 객체를 가져온다. (new 기본생성자())
    5) setProperty : <jsp:setProperty />
        자바 빈즈의 프로퍼티 값을 설정한다(setter)
    6) getProperty : <jsp:getProperty />
        자바 빈즈의 프로퍼티 값을 가져온다(getter)
        
* JavaBeans : 자바 객체, jsp의 표준 액션 태그로 접근하는 자바 클래스

3. 내장 객체(Internal Object)
    - 객체화를 하지 않고 사용할 수 있는 객체
    - jsp가 서블릿으로 변환될 때
      웹 컨테이너가 메모리에 할당
    
    request : 웹 브라우저의 요청 정보 저장
    response : 웹 브라우저의 요청에 따른 응답 정보 저장
    out : jsp에 출력할 내용 정보 저장
    session : 웹 브라우저의 정보 저장을 위한 세션
    config : jsp에 대한 설정 정보 저장
    pageContext : jsp에 대한 정보 저장
    exception : 예외 발생 시 사용하는 객체

    쿠키 : 특정 사이트에 대한 정보를 내 pc에 저장하고 있는 것

4. Cookie
(1) 정의 
    웹 브라우저가 보관하고 있는 데이터
    웹 서버에 요청을 보낼 때 쿠기를 헤더에 담아 전송
    웹 브라우저는 쿠키가 삭제되기 전까지 웹 서버에 쿠기 전송

(2) 장점과 단점
    클라이언트의 특정 폴더에 저장하므로 서버의 부하를 줄임
    정보가 사용자 PC에 있기 때문에 보안의 위협 존재
    데이터 저장 용량의 한계가 있음
    일반 사용자가 브라우저 내부의 기능인 "쿠기 차단"을
    사용하면 무용지물됨

(3) 동작 방식
    1) 쿠키 생성 : 웹 서버에서 쿠키를 생성하고
                  응답에 쿠키를 담아 웹 브라우저로 전송
    2) 쿠키 저장 : 웹 브라우저는 응답 데이터에 있는 쿠키를
                  메모리 또는 파일에 저장
    3) 쿠키 전송 : 웹 브라우저는 쿠기 요청이 있을 때 마다
                  웹 서버에 전송하고 웹 서버는 쿠키를 활용

5. Session
(1) 정의
    내장 객체로서 브라우저마다 한 개씩 존재하고
    고유한 세션 ID 생성 후 정보를 추출

(2) 장점과 단점
    JSP에서만 접근하므로 보안성이 좋음
    저장 용량의 한계 없음

    * 쿠키보다 세션이 더 안정적이고 안전하나
      세션을 저장할 공간이 부족할 경우 쿠키 사용
