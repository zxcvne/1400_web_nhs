2. CGI(Common Gateway Interface)
(1) CGI 이전 웹서버 : 사용자가 특정 경로를 접근하면 리소스 제공
(2) CGI : 사용자의 입력을 인자(파라미터)로 입력 받아
          프로그램을 동작시켜 실행결과를 웹으로 제공
          웹 서버에서 동적인 페이지를 보여주기 위해
          프로그램을 구동하는 기술
(3) CGI 동작 구조
    1) User의 요청(request)이 웹 서버로 전달
    2) 웹 서버는 요청에 있는 주소가 CGI 프로그램에 매핑되어 있는지 확인
    3) 매핑이 없으면 오류, 매핑 성공 시 실행
    4) 웹 서버는 CGI 프로그램이 표준 출력으로 보낸 내용을
       사용자에게 돌려준다.

    sns 로그인 : oAuth
    static web page : 기업 소개 페이지
    dynamic web page : 소셜 네트워크 페이지
                       데이터가 동적이며 사용자의 행동에 민감하게 반응
                       추천 알고리즘, 팔로우, 좋아요

3. 서블릿(Servlet) 
    - 서버(Server) + 애플릿(Applet) 
    - Java 웹 서버로 웹 페이지를 제공할 때 동적 데이터 제공을 도와주는 역할
    - 사용자님 반갑습니다 -> 홍길동님 반갑습니다

(1) 프로세스
    웹 브라우저에서 Request를 보내면 Response가 돌아옴
    웹 브라우저에서 Request -> Servlet -> Response

(2) 문자열의 해석(parsing)에 들어가는 시간을 줄이고
    비즈니스 로직의 개발에 집중 가능

3. 서블릿과 CGI의 차이점
    - CGI는 브라우저에서 웹 서버로 요청이 가면 각각의 프로세스 생성
        -> 대량 접속 시 부하 발생
    - 서블릿은 프로세스 자체가 1개
      요청 발생 시 내부의 Thread를 사용
      Thread의 개수를 전체적으로 관리
